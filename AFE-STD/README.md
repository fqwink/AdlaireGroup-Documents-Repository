## プロジェクト概要

AFE Standard（AFE-STD）は、Adlaire Framework Ecosystem（AFE）が採用する**独自の標準仕様・コーディング規則を策定・管理する独立プロジェクト**である。

AFE は外部標準（PHP-FIG PSR 等）への依存を排し、AFE-STD を唯一の標準体系として全面採用する。
AFE-STD は AFE プロジェクトから独立して管理されることで、標準仕様の独立した進化・バージョン管理を可能にする。

### 位置づけ

```
AdlaireGroup-Documents-Repository/
├── AFE/          ← フレームワーク実装プロジェクト
│                    AFE-STD に準拠して実装される
└── AFE-STD/      ← 標準仕様プロジェクト（本プロジェクト）
                     AFE の標準・規則類を独立して定義・管理
```

### 目的

1. **標準の独立管理**: フレームワーク実装（AFE）と標準仕様（AFE-STD）を分離し、それぞれが独立してバージョン管理される体制を確立する
2. **一貫性の確保**: Adlaire Group が関与するすべてのプロジェクトにおいて、統一された命名規則・コーディング規約を適用する
3. **PSR からの完全独立**: PHP-FIG PSR への依存を排し、AFE の設計思想に最適化された標準仕様を自律的に定義する
4. **将来拡張性**: Go 言語との統合（ハイブリッド構造）を見据えた言語非依存の標準設計

## AFE-STD カテゴリ体系

| カテゴリ | 番号帯 | 対象領域 |
|---------|--------|---------|
| **コア標準** | 1xx | 命名規則・オートローディング・コーディングスタイル |
| **HTTPレイヤー標準** | 2xx | HTTPメッセージ・ファクトリー・ミドルウェア・クライアント |
| **コンテナ・DI標準** | 3xx | 依存性注入・サービスコンテナ |
| **ロギング・イベント・時刻標準** | 4xx | ロガー・イベントディスパッチャー・クロック |
| **キャッシュ標準** | 5xx | キャッシュIF・シンプルキャッシュ |
| **アプリケーション標準** | 6xx | ORM・認証認可・バリデーション・CLI・Go連携 |
