## 1. プロジェクト概要

### 1.1 プロジェクト名

**AFE Standard**（以下、「本プロジェクト」または「AFE-STD」）

### 1.2 プロジェクトの位置づけ

本憲章は、AFE Standard プロジェクトに関する最上位ドキュメントである。
本憲章に基づいて、すべての関連標準仕様書・規則書が作成・管理されるものとする。

本プロジェクトは **Adlaire Framework Ecosystem（AFE）プロジェクトから独立した** 標準仕様管理プロジェクトとして位置づけられる。

### 1.3 プロジェクト概要

AFE Standard（AFE-STD）は、Adlaire Framework Ecosystem（AFE）が採用する独自の標準仕様・コーディング規則を策定・管理するプロジェクトである。

AFE は外部標準（PHP-FIG PSR 等）への依存を排し、AFE-STD を唯一の標準体系として全面採用する。
AFE-STD はフレームワーク実装（AFE）とは独立して管理され、それぞれが独立したバージョン管理・ライフサイクルを持つ。

## 2. プロジェクト目的・背景

### 2.1 背景

Adlaire Framework Ecosystem（AFE）は、PHP 8.2 以上を採用した独自フレームワークであり、Adlaire Group が関与するすべてのプロジェクトの共通基盤として機能する。

AFE の開発・運用において、以下の課題が設計段階から明確化された。

- PHP-FIG PSR 標準は汎用性を重視するがゆえに、多くの定義が「推奨（SHOULD）」に留まる
- PSR が規定しない領域（命名規則の詳細・型宣言ルール・コンポーネントパターン命名等）が広範に存在する
- 外部標準に依存することで、AFE 固有の設計思想・アーキテクチャパターンを標準として表現できない

### 2.2 目的

本プロジェクトは、上記課題を解決するため、以下を目的とする。

1. **PSR からの完全独立**
   PHP-FIG PSR に依存せず、AFE の設計思想・技術スタックに最適化された標準仕様を自律的に定義・管理する。

2. **標準の一元化**
   命名規則・コーディングスタイル・各種インターフェース規約をすべて AFE-STD として一元管理し、Adlaire Group 内での標準の分散・矛盾を排除する。

3. **フレームワークと標準の独立管理**
   AFE（実装）と AFE-STD（標準）を独立したプロジェクトとして管理することで、標準仕様の改訂がフレームワーク実装に依存しない体制を確立する。

4. **将来の言語拡張への対応**
   Go 言語との統合（ハイブリッド構造）を見据えた、言語非依存の標準インターフェース設計を推進する。

### 2.3 ビジョン

> **「AFE のすべてのコードは、AFE-STD によって定義された一貫した標準のもとに構築される。」**
