## バージョン管理履歴規則

本ファイルは Adlaire Framework Ecosystem（AFE）プロジェクトに関わる**すべてのファイルの変更履歴を一元管理**するファイルである。
各ファイルに個別の改訂履歴セクションは設けず、本ファイルに集約する。

| 規則 | 内容 |
|------|------|
| **バージョン表記** | `Ver.{メジャー}.{マイナー}-{リビジョン}` 形式を必須とする（詳細: `VERSIONING.md`） |
| **リビジョンリセット** | ❌ 禁止。プロジェクト開始からの総累積管理とする |
| **記録単位** | ファイルごとにセクションを立てて管理する |
| **改訂者** | 変更を行った権限保有者の氏名・役職を明記する |
| **改訂内容** | 変更の要点を簡潔に記載する |
| **追記順序** | 各ファイルセクション内で最新エントリを**末尾**に追記する |

> 📖 バージョン表記の詳細規則は [`VERSIONING.md`](./VERSIONING.md) を参照すること。

---

## ⛔ 改訂履歴の記載に関する禁止事項

> **本ファイル（`VERSION_HISTORY.md`）は、AFE プロジェクト配下のすべてのドキュメントの改訂履歴を一元管理する唯一の正規ファイルである。**

| 規則 | 内容 |
|------|------|
| **改訂履歴テーブルの記載場所** | `VERSION_HISTORY.md` のみに記載する。それ以外のファイルへの記載を **禁止** する |
| **各ドキュメント内への改訂履歴テーブル記載** | ❌ **禁止** — `README.md`・`FRAMEWORK_SPECIFICATION.md`・`PROJECT_CHARTER.md`・`VERSIONING.md` 等、いかなるドキュメントにも改訂履歴テーブルを設けてはならない |
| **各ドキュメント内での改訂履歴セクション** | ❌ **禁止** — `## 改訂履歴`・`## 変更履歴`・`## 変更履歴サマリー` 等の見出しセクションをドキュメント内に設けてはならない |
| **違反時の対応** | 発見次第、該当セクションを即時削除し、本ファイルへ統合する |

> 💡 **理由**: 改訂履歴を各ドキュメントに分散させると、管理の重複・不整合・記録漏れが生じる。`VERSION_HISTORY.md` への一元集約により、全ドキュメントの変更履歴を確実かつ一貫して追跡できる。
